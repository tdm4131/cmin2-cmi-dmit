## CMIN2 和 CMI 的本质区别，一段话说完

CMI（AS58453）是中国移动国际有限公司的通用国际出口线路，面向全球提供常规跨境连接。CMIN2（AS58807）是 CMI 在此之上单独建设的"国际精品网"，定位跟电信 CN2 GIA、联通 AS9929 同一梯队——专用通道、独立出海路径、更高 QoS 优先级。

简单说：CMI 是普通通道，CMIN2 是快车道。同样是移动运营的线路，走的物理路径和拥塞控制逻辑完全不一样。

---

## 两条线路的核心差异在哪里

**ASN 不一样，这是判断最直接的依据：**

- 普通 CMI：AS58453
- CMIN2 精品网：AS58807

用 traceroute 或 mtr 跑一下路由，回程经过的 ASN 里看到 58807，才是真正的 CMIN2。如果只显示 58453，走的还是普通移动国际线路。这个方法在买 VPS 之前可以用商家提供的测试 IP 先跑一遍，验证一下。

**延迟和丢包的差距实测有多大？**

普通 CMI 在晚高峰（北京时间 20:00-23:00）往往出现 3-5% 丢包，洛杉矶到上海的延迟会漂到 180-210ms 区间。CMIN2 在同一时间段，美西到上海的延迟能压在 125-145ms 左右，丢包率长期低于 1%。差的不是一点点。

**入境方式也不同。**

CMIN2 的流量路径大致是：海外 PoP（洛杉矶/东京/新加坡等 223.120.* 段）→ AS58807 骨干 → 国内登陆点（上海/广州/北京）→ AS9808 省网 → 用户。中间绕开了大量公共交换节点，所以拥塞少。普通 CMI 走的是公共骨干，高峰期跟着大家一起堵。

## 两者的使用场景怎么选

说实话，CMIN2 并不是所有人都需要。

**适合 CMIN2 的场景：** 用户主体是移动宽带，对延迟敏感（游戏服务器、实时音视频、高频接口调用），或者业务需要在晚高峰保持稳定响应的。

**普通 CMI 够用的场景：** 主要做静态资源分发、邮件服务、不依赖实时性的后台任务。晚高峰偶尔慢一些，但长期跑没问题。

还有一种情况——如果你用的是香港机房的 Eyeball 系列（比如 DMIT 香港 EB），走的是 CMI 回程，不是 CMIN2。这两个容易混淆，CMI 在香港机房的表现因为物理距离近，实际体验跟 CMIN2 差距没那么大，但本质上还是两条不同的线路。

## FAQ

**Q：怎么验证我买的 VPS 是不是真的走 CMIN2？**

A：最简单的方法是在国内用移动宽带 mtr 打过去，或者在 VPS 上跑 backtrace 查回程路由。回程链路里出现 AS58807，就是 CMIN2；如果只看到 AS58453，走的是普通 CMI。买之前可以先用商家提供的测试 IP（DMIT 洛杉矶 EB 的测试 IP 是 154.17.226.2）跑一遍确认。

**Q：CMIN2 对电信和联通用户也有帮助吗？**

A：有，但程度不同。CMIN2 本身是移动的精品线路，对移动用户效果最直接。DMIT LAX.EB 的线路方案是电信联通回程走 AS9929，移动走 CMIN2，所以电信联通用户享受的是 9929 优化，不是 CMIN2。如果你主要是电信联通用户，选 EB 一样有优化，只是走的不是 CMIN2 那条。

**Q：LAX.EB 和 LAX.Pro 值不值得付差价？**

A：这取决于你的实际需求。EB 系列电信联通回程走 9929，移动走 CMIN2，在大多数使用场景下已经够用，晚高峰表现稳定。Pro 系列三网全走 CN2 GIA，延迟更低、稳定性更好，但价格贵一倍多。如果你的业务对延迟非常敏感（游戏服务器、实时通话），或者用户都在电信专线，Pro 值得考虑。普通建站、代理、跑服务，EB 完全够。

**Q：DMIT 香港 EB 和洛杉矶 EB 的区别？**

A：机房位置和线路方案都不同。香港 EB 走 CMI 回程（AS58453），不是 CMIN2，但因为地理上离大陆近，延迟本来就低，CMI 的体验跟 CMIN2 差距没洛杉矶那么明显。洛杉矶 EB 走 AS9929 + CMIN2 回程，地理距离更远但线路优化到位。价格上香港 EB 贵不少，$179.9/年起，洛杉矶 EB 年付 $39.9 起。
